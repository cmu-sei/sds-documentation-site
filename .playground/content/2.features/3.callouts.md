# Callouts

::callout{type="subtle" variant="green"}
✅ **Note:** This documentation about callouts **is real** unlike the rest of the site that uses dummy content.
::

Use **Callouts** to highlight key information, guide readers’ attention, or emphasize related content on a page. This page demonstrates how to use the different **variants** (color styles) and **types** (visual emphasis) of the [**SDS Callout**](https://designsystem.sei.cmu.edu/components/callout) component.

---

## Configuration Options

### Type
Defines the **level of visual emphasis**.

- **`bold`** — Use when the message is critical or requires high visual emphasis.  
- **`subtle`** — Use for secondary messages that still need to stand out.  
- **`outline`** — Use for minimal emphasis, often for inline tips or supporting info.  

### Variant
Determines the **color meaning** or semantic purpose.

- **`blue`** — Tips, help messages, or general information.  
- **`green`** — Success, confirmation, or positive outcomes.  
- **`orange`** — Warnings, cautions, or potential issues.  
- **`red`** — Errors, failures, or critical blocking problems.  
- **`gray`** — Neutral, supporting, or reference info.  
- **`purple`**, **`indigo`**, **`teal`** — Branded or custom callouts for feature announcements or highlights.  

### Size
Controls the **padding and visual density** of the callout.

- **`sm`** — Smaller padding and font size; best for compact layouts.
- **`md`** — Default size; balanced spacing for general use.

## 🔷 Blue – Informational Notes

Use the **blue** variant to share background context, helpful hints, or additional explanations that aid reader understanding.  

### Outline

::callout{variant="blue" type="outline" size="sm"}
🌟 **Info:** Use this format to provide context or definitions that help clarify a topic without interrupting the main flow.
::

### Subtle

::callout{variant="blue" type="subtle"}
💡 **Tip:** When referencing internal systems, always include the full acronym on first mention to help new readers understand.
::

### Bold

::callout{variant="blue" type="bold"}
🕚 **Note:** This section is periodically updated as the design system evolves. Check the version history for recent changes.
::

## ✅ Green – Positive Information

Use the **green** variant for highlighting helpful outcomes, confirming good practices, or reinforcing recommended approaches.  
It draws attention to “what works well” or validated methods.

### Outline

::callout{variant="green" type="outline"}
🌿 **Tip:** Consider documenting examples from real projects to help illustrate how a component is used successfully.
::

### Subtle

::callout{variant="green" type="subtle"}
✅ **Good Practice:** Keep each documentation page focused on one concept or component to make information easier to find.
::

### Bold

::callout{variant="green" type="bold"}
🏆 **Recommended:** Teams that standardize on shared documentation templates reduce onboarding time by up to 40%.
::

## 🍊 Orange – Warnings

Use the **orange** variant to highlight potential confusion, common mistakes, or critical editorial cautions.  
It helps readers avoid misinterpreting or misusing the information.

### Outline

::callout{variant="orange" type="outline"}
⚡ **Heads-up:** Some diagrams or screenshots may depict earlier versions of the interface. Always confirm component styles against the current release.
::

### Subtle

::callout{variant="orange" type="subtle"}
⚠️ **Caution:** Avoid duplicating design tokens in code samples. Reference them directly from the shared design system package.
::

### Bold

::callout{variant="orange" type="bold"}
🚧 **Important:** Deprecated components should not be documented as “current.” Move them to the Legacy section to prevent confusion.
::

## ❤️ Red – Critical Information

Use the **red** variant for major documentation issues or content that requires immediate attention.  
It should be used sparingly and only for serious errors or blockers.

### Outline

🧯 **Reminder:** Remove placeholder text and dummy data before publishing any public-facing documentation.
::

### Subtle

::callout{variant="red" type="subtle"}

::

### Bold

::callout{variant="red" type="bold"}

::
